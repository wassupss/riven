Agent-native Workflow

에이전트를 옆에 두는 게 아니라, 워크플로 한가운데에 둡니다

대부분의 IDE에서 AI는 사이드바에 얹힌 채팅입니다. riven의 터미널은 실제 에이전트 자식 프로세스를 지켜봅니다 — 출력 텍스트 추측이 아니라 프로세스 기준으로 실행 상태와 attention(벨, 작업 완료)을 판별하고, 편집이 끝나면 리뷰까지 이어집니다.

01 — 보낸다

Everything an IDE should be

에이전트만 특별한 게 아닙니다. IDE로서도 진짜입니다

Electron + React + TypeScript로 만든 데스크톱 앱. 에디터 코어부터 터미널, 언어 지원까지 — 매일 쓰는 도구가 갖춰야 할 것들을 제대로 갖췄습니다.

Monaco 에디터

VS Code와 동일한 에디터 코어. 파일 URI로 키잉된 여러 모델을 하나의 에디터가 다루는 구조라 탭 전환이 즉각적입니다.

죽지 않는 터미널

xterm.js + node-pty. PTY가 main 프로세스에 상주해 ⌘R과 패널 재마운트에도 살아남고, 직렬화된 화면 스냅숏으로 그대로 재접속합니다.

에이전트 인식 터미널

출력 문자열 추측이 아니라 실제 에이전트 자식 프로세스에서 실행 상태를 읽습니다. 벨·작업 완료 같은 attention도 프로세스 기준으로 감지합니다.

멀티 언어 LSP

완성·hover·정의 이동·진단. TypeScript/JS, Python(Pyright), YAML, Bash는 설정 없이 내장. clangd·gopls·rust-analyzer는 PATH에 있으면 자동으로 켜집니다.

dockview 도킹

VS Code식 유연한 도킹. 탐색기·에디터·프리뷰·터미널 전부가 드래그 가능한 패널이고, 어느 것이든 별도 창으로 팝아웃됩니다.

워크스페이스 세션

프로젝트를 여러 개 동시에 열고, 각각 고유한 탭·레이아웃·터미널을 유지합니다. 재시작해도 레이아웃과 열린 탭이 그대로 복원됩니다.

Shiki 하이라이팅

TextMate급 정밀 하이라이팅. tsx/jsx를 일급으로 지원해, VS Code에서 보던 그 색 그대로 읽힙니다.

멀티 프로바이더 AI

원하는 AI 프로바이더를 골라 쓰고, 사용량을 한눈에 추적합니다. 커스텀 키바인딩과 테마도 함께.
